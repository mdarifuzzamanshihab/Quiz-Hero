1. What is the difference between map, filter, find and foreach?

Ans: 'map()' executes the same code on every element in an array and returns a new array with the updated elements. 'filter()' checks every element in an array to see if it meets a certain criteria and returns a new array with the elements that return truthy for the criteria. 'find()' returns only one match in an array. If in an array is more than one result, the function will return the first that has matched. 'forEach()' is used to execute the same code on every element in an array but does not change the array and it returns undefined.

2. What's the difference between Local Storage, Session Storage, and Cookies?

Ans: The stored data in local storage is not deleted when the browser is closed. On the other hand session storage data is stored only for the session and will be deleted when the browser is closed. And finally Cookies data can be set to expire at a certain time.

3. What is the event loop in JavaScript and how does it work?

Ans: The event loop is the secret behind JavaScriptâ€™s asynchronous programming. JS executes all operations on a single thread, but using a few smart data structures, it gives us the illusion of multi-threading.

In JavaScript, asynchronous functions are sent to the browser API for execution. The API then sends the function back to the event queue, which is continuously monitored by the event loop. The event loop adds functions from the event queue to the call stack.When the call stack is empty, the event loop adds the next function from the event queue to the call stack and executes it. This allows for asynchronous and non-blocking code execution, and ensures that the user interface remains responsive.
